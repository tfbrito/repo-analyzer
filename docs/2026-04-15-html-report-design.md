# HTML Report Generation — Design Spec

## Context

Feature para o repo-analyzer: gerar uma pagina HTML apelativa (Gradient Dark) junto com o relatorio Markdown existente. Template fixa que parseia o MD e injeta seccoes. HTML + CSS separado, style.css partilhado.

## Overview

Novo modulo `html_report.py` que:

1. Recebe o Markdown do relatorio (string) e o nome do repo
2. Parseia as 6 seccoes conhecidas (split por `## N.`)
3. Converte cada seccao de MD para HTML com a library `markdown` (extensao `tables`)
4. Injeta o conteudo numa template HTML com estrutura semantica (header, metrics, cards)
5. Retorna o HTML completo

O CSS (Gradient Dark) fica como ficheiro separado `style.css` no package, copiado para `output/` na primeira geracao.

## Module: html_report.py

### Functions

- `generate_html(markdown_content: str, repo_name: str) -> str`
  - Entry point. Parseia o MD, gera HTML completo.
  - Retorna string HTML pronta a escrever em ficheiro.

- `get_stylesheet() -> str`
  - Retorna o conteudo do style.css embebido no package.
  - Usa `importlib.resources` ou `Path(__file__).parent / "style.css"` para ler o ficheiro.

- `_parse_sections(markdown_content: str) -> dict[str, str]`
  - Split do Markdown pelas 6 seccoes usando regex no padrao `## N. Title`.
  - Retorna dict com keys: "summary", "tech_stack", "structure", "quality", "improvements", "security".
  - Cada value e o conteudo MD da seccao (sem o header).

- `_md_to_html(md_content: str) -> str`
  - Converte uma string Markdown para HTML usando `markdown.markdown()` com extensao `tables`.

- `_build_html(repo_name: str, sections: dict[str, str]) -> str`
  - Constroi o documento HTML completo com a template.
  - Cada seccao fica num `<section class="card">` com numero e titulo.
  - Header com nome do repo.
  - Footer com "Generated by repo-analyzer".

### Parsing Strategy

O relatorio tem sempre 6 seccoes com headers `## 1.` a `## 6.`:
1. Project Summary
2. Tech Stack
3. Structure & Architecture
4. Code Quality
5. Suggested Improvements (Top 5)
6. Security Issues (Top 5)

Regex para split: `r'^## \d+\.\s+'` com flag MULTILINE.

Cada seccao e convertida independentemente de MD para HTML com `markdown.markdown(content, extensions=["tables"])`.

## CSS: style.css

Tema Gradient Dark conforme mockup aprovado:

**Design tokens:**
- Background: gradiente `#0f172a` a `#1e293b`
- Cards: `#1e293b` com border `#334155`, border-radius 12px
- Texto principal: `#e2e8f0`, secundario: `#94a3b8`
- Accents: cyan `#22d3ee`, purple `#a78bfa`
- Severidade: High `#f87171`, Medium `#fbbf24`, Low `#94a3b8`
- Font: `-apple-system, system-ui, sans-serif`
- Max-width: 900px centrado

**Componentes:**
- `.container` — wrapper principal
- `.report-header` — nome do repo, descricao, badges
- `.metrics` — grid de 2 colunas com contadores
- `.card` — seccao com background e border
- `.section-num` — numero da seccao com gradiente
- `.badge`, `.badge-cyan`, `.badge-purple` — pills para tech stack
- `.severity-critical`, `.severity-high`, `.severity-medium`, `.severity-low` — badges de severidade
- `table`, `th`, `td` — tabelas estilizadas
- `pre`, `code` — blocos de codigo
- `footer` — rodape

## Integration in main.py

Apos escrever o .md, gerar o HTML e copiar o CSS:

```python
from .html_report import generate_html, get_stylesheet

# After writing MD report:
html_path = output_path.with_suffix(".html")
html_content = generate_html(report, repo_name)
html_path.write_text(html_content)

css_path = output_path.parent / "style.css"
if not css_path.exists():
    css_path.write_text(get_stylesheet())
```

A escrita do HTML fica dentro do mesmo try block que o MD.

Output apos run:
```
output/nocode-analysis.md
output/nocode-analysis.html
output/style.css
```

## Scope

### Included
- Parser das 6 seccoes do relatorio
- Template HTML Gradient Dark
- CSS partilhado em style.css
- Geracao automatica em cada run
- Badges de severidade com cores

### Excluded (YAGNI)
- Dark/light mode toggle
- JavaScript interativo
- Abrir o browser automaticamente
- Responsividade avancada (alem do max-width basico)
